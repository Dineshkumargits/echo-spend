●  Echo Spend · Design Direction 01

Money as signal.

Every rupee you move announces itself — a bank SMS lands, the on-device AI listens, you confirm with a swipe. No other finance app owns this loop, so the design language comes from it: pulses, echoes, sweeps. Not borrowed from any neobank.

The idea

The app already has a metaphor.
It's in the name.

Echo Spend's entire loop is acoustic: a transaction emits a signal, the app hears it, an echo comes back for you to confirm. The visual system makes that loop visible — so the product's real differentiator (on-device AI that listens privately) becomes the thing you see, not a settings toggle you read about.

〰️ Signal in

Spending emits a pulse

Debits render in pulse amber — warm, outgoing energy. Unreviewed transactions carry a live heartbeat dot. The SMS itself appears as a raw signal trace in monospace.

📡 AI listens

Scanning is a sonar sweep

SmartScan stops being a progress bar and becomes a radar: a sweep arm rotates over your inbox, blips light up as transactions are found. The private, on-device model finally gets a face.

💠 Echo back

Money in returns as an echo

Credits answer in echo aqua — cool, returning. Confirming a transaction triggers a single ripple from your thumb. Goal milestones resonate: rings expand outward.

Color system

Signal green, amber pulse, aqua echo.

One deliberately biased neutral (a teal-black, not #000), a signal-green brand accent carried over from the original logo, two semantic signal hues, and one alert red reserved for genuinely bad news. This kills the current iOS-clone palette and the ThemeProvider/Tailwind mismatch in one move — every value below is a token with a dark and a light resolution.

Amber for debits, not red. Spending is normal, not a failure state — red screams at users dozens of times a day in every other tracker. Amber reads as "energy spent"; red stays meaningful for actual problems.

Typography

Numerals are the interface.

A finance app's real typography is its numbers. The system: a tight grotesk display for hero amounts, a humanist text face for everything readable, and a monospace for raw signals — SMS traces, hashes, timestamps — so "machine data" is visually distinct from "your money".

Key screens

The system on glass.

Same navigation, same data, same business logic — every screen keeps its current structure and store/DB wiring. Only the skin, hierarchy and motion change.

Motion & feel

Everything moves like sound.

Moti and Reanimated are already in the project — the redesign choreographs them instead of adding libraries. Four rules keep it coherent:

Rule 01 · Emit

Actions ripple outward

Confirm, pay, contribute — anything that moves money emits one ring from the touch point, synced with a medium haptic. Never more than one ring per action.

Rule 02 · Arrive

New data fades up, staggered

List items arrive like echoes returning: 40ms stagger, small y-offset, no bounce. Springy overshoot is reserved exclusively for celebrations.

Rule 03 · Breathe

Only live things pulse

The heartbeat animation is earned: unconfirmed transactions, an active scan, a due bill. If nothing needs the user, the interface is perfectly still.

Rule 04 · Resonate

Milestones ring

Goal reached, budget cycle survived, debt cleared — concentric rings expand across the card with a success haptic. The one moment the app is allowed to show off.

Rollout

Four phases, zero logic changes.

Each phase ships independently and the app stays fully working between them. All changes live in the presentation layer — services, store logic, and the database are untouched.

Phase 1

Token foundation

One source of truth for color, type, spacing, radius and motion durations, feeding both ThemeProvider and Tailwind (fixing their current mismatch). Load the three typefaces via expo-font. The whole app shifts palette in a day.
